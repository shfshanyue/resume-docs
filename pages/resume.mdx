在当今竞争激烈的就业市场中，一份优秀的简历可以成为你获得理想工作的关键。本指南将帮助你创建一份引人注目、结构清晰的简历，突出你的优势，增加获得面试机会的可能性。

## 简历结构

一份标准的简历通常包含以下几个部分：

1. **个人信息**
    - 姓名、电话、（微信）、电子邮件、所在城市
    - 可选：GitHub 个人主页、个人网站或技术博客
2. **个人技能**
    - 列出与目标职位相关的核心技能
    - 如果项目经历丰富，可以考虑将此部分融入到具体项目描述中
3. **开源贡献**（如适用）
    - 如果有丰富的开源经历，可以单独列为二级标题
    - 突出你的贡献和影响力
4. **教育经历**
    - 学校、专业、学位、毕业时间
    - 相关的荣誉奖项或重要课程项目
5. **工作经历**
    - 按时间倒序排列
    - 公司名称、职位、工作时间
    - 突出你的成就和贡献，而不仅仅是职责描述
6. **项目经历**
    - 重点描述与目标职位相关的项目
    - 突出你的角色、使用的技术和取得的成果

## 简历撰写技巧

1. **保持简洁**：简历篇幅控制在1-2页内，突出最相关和最重要的信息。
2. **量化成果**：尽可能使用数据和具体例子来展示你的成就。
3. **关键词优化**：根据目标职位的要求，适当使用行业关键词。
4. **格式一致**：保持字体、间距和排版的一致性，提高可读性。
5. **突出重点**：将最相关和最有说服力的信息放在每个部分的开头。

## GitHub 与技术博客

GitHub 和技术博客是展示你专业能力的重要平台，它们能够：

1. **证明编码能力**：通过开源项目展示你的代码质量和技术深度
2. **展示学习热情**：持续的提交记录和博客更新反映你对技术的热爱
3. **技术影响力**：优质的技术文章和项目可以扩大你的职业影响力

根据招聘数据显示，HR 平均只会花费 3-7 秒浏览一份简历。在这短暂的时间内，以下"硬性指标"最容易抓住招聘方的注意：

- 大厂工作背景
- 教育背景
- 活跃的 GitHub 贡献记录
- 高质量的技术博客

这些客观指标能显著提升你获得面试机会的概率，而 Github/Blog 的重要性不亚于大厂工作背景及优秀的教育背景。

目前大部分人简历的项目经验部分千篇一律，过于雷同，如果你没有 985/211 的学历背景以及大厂工作背景，想要在众多简历中脱颖而出，GitHub 和技术博客是展示你专业能力的重要途径。如果你准备在接下来的三个月内跳槽，那么你完全有充足的时间来准备你的 GitHub 和博客。

如果你没有大厂背景和优秀的教育背景，那么以下两点建议可以帮你快速提升面试概率：

1. 每天浏览 GitHub 发现优秀项目及代码，持续 30 天，使你的 GitHub 时间线更加活跃，并优化个人资料（GitHub Profile）页面。
2. 坚持写博客，确保最近一个月内发布 5 篇有深度的文章，而不仅仅是简单的知识整理。

Blog 可以参考 [技术博客运营指南](./blog)。

## 简历不提薪资

不要将期望薪资放在简历中。

在求职过程中，薪资是一个敏感而重要的话题。以下是一些处理薪资问题的建议：

- 技术面试官（一面、二面）通常不负责薪资相关事宜，过早透露薪资期望可能影响面试评估的客观性，也有可能因薪资倒挂而失去面试机会。
- 建议在 HR 面试前谨慎处理薪资话题。
  - 如果 HR 在初筛阶段询问薪资预期，可采用"薪资范围"策略：例如：实际期望 25-30k 的情况下，可以回答"25k+"。避免一开始就说 30k 因期望过高而失去面试机会。
  - 最终谈薪环节时，可以先提出较高预期（如 30k），如果谈薪失败，还可以在可接受范围内下调（如 25k）。

## 结语

一份优秀的简历是展示自己价值的重要工具。通过精心设计简历结构，突出你的优势和成就，你将大大提高获得理想工作的机会。

记住，简历是你的个人营销文档，要**根据不同的职位和公司适当调整**。同时，在处理薪资问题时保持策略性和灵活性，这将有助于你在求职过程中占据有利位置。
